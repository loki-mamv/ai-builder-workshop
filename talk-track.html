Slides 8-10 — The AI Builder Landscape / When to Use What

Say: "Let me orient you to the landscape. There's a spectrum:"

  • No-code (Bolt, Lovable, v0): "Point and click, get a working app. Great for prototypes and simple tools. Ceiling is low but the floor is zero."
  • AI-assisted coding (Cursor, Windsurf, GitHub Copilot): "You're in an IDE, but AI is writing 80% of the code. Need some technical comfort."
  • AI agents (Claude Code, Codex): "You give it a task, it figures out the implementation. This is where I live. You need to understand what you're asking for, but you don't need to know how to code it."

Key insight: "Don't pick one and stick with it. I use all of them depending on the task. Quick prototype? Bolt. Complex app? Claude Code. The tool doesn't matter — the thinking does."

THEME 2: FRONT-LOAD THE THINKING (20 min)

Goal: This is the meat. Show the actual process with a live example. This is where they learn.

Slides 11-14 — The Modern Stack / Tool Map / How It Connects

Say: "Before I show you the process, let me quickly map the tools. You don't need all of these — you need maybe 3."

  • Frontend: "What users see. HTML, React, whatever. AI handles this."
  • Backend/Database: "Where data lives. Supabase is my go-to — it's like Firebase but simpler."
  • Hosting: "Where it lives on the internet. GitHub Pages for static, Vercel for apps. Both free."
  • AI layer: "Claude, GPT, Gemini — the brains. Connected via API."

"You don't need to memorize this. You need to know it exists. When you tell Claude Code 'I need a web app with a database,' it knows how to wire all of this together."

Slides 15-16 — My Process / The Spec

THIS IS THE KEY SLIDE. Slow down here.

Say: "Here's the single most important thing I'll tell you today: the quality of your spec determines the quality of your output."

"Most people open ChatGPT and type 'build me a CRM.' That's like walking into a contractor's office and saying 'build me a house.' What kind of house? How many rooms? What style? Where?"

"The spec conversation is where you do your real work. Here's what a good spec includes:"

  1. What problem does this solve? — "Not 'what do I want to build' but 'what pain am I eliminating?'"
  2. Who uses it? — "Be specific. 'Sales reps who need to log calls in under 30 seconds.'"
  3. What does success look like? — "User can do X in Y clicks/seconds."
  4. What's the MVP? — "What's the smallest version that's still useful? Build that first."
  5. What does it look like? — "Show examples. Screenshot a competitor. Sketch on a napkin."

Slides 17-18 — The Conversation / A Real Example

🔴 LIVE DEMO TIME

Say: "Let me show you what this actually looks like. I'm going to build something right now, live, in front of you."

Demo options (pick one based on audience):

  • Option A: Simple tool — "Let's build a meeting cost calculator. You put in number of people, their hourly rates, meeting duration — it tells you how much that meeting just cost your company."
  • Option B: Something from the audience — "Actually, let me ask — does anyone here have a simple tool they wish existed? Something that would save you 30 minutes a day?" [Take a suggestion, build it live.]
  • Option C: GenHealth-relevant — "Let me build a prior authorization status tracker. A simple dashboard where you can see where every PA stands."

Walk through the conversation with Claude Code:

  1. Start with the spec — type it out conversationally
  2. Show Claude Code generating the first version
  3. Open it in the browser — "Look, it works. 3 minutes."
  4. Iterate: "Actually, let's add a dark mode. Let's make the chart interactive."
  5. Show the refinement loop — 2-3 iterations max

Key line while demo runs: "Notice I'm not writing code. I'm having a conversation. I'm describing what I want in plain English. The AI writes the code. I judge the output. That's the workflow."

Slides 19-20 — First Build & Refinement

Say: "What you just saw is the real loop: Spec → Build → Look at it → Refine → Repeat."

"The first build gets you 80-90% there. The iteration is where you polish. And here's the thing most people don't realize: each iteration takes minutes, not days."

Slides 21-22 — Common Pitfalls & Best Practices

Pitfalls to call out:

  • "Vague specs get vague results. 'Make it better' is not feedback. 'Make the header larger and change the green to blue' is."
  • "Don't try to build everything at once. Ship the MVP. Then add features."
  • "Don't trust AI blindly. Review the output. Test it. AI makes confident-sounding mistakes."
  • "Don't skip the security basics — especially if you're handling user data." [Point to appendix slides]

THEME 3: YOU CAN BUILD THIS (15 min)

Goal: Give them the concrete tools and confidence to start today.

Slides 23-29 — Claude Code Power Features (MCP, CLIs, CLAUDE.md, etc.)

Say: "I'm going to go a little deeper on Claude Code specifically because it's what I use every day. You don't need to memorize any of this — the slides will be shared."

Hit the highlights fast:

  • MCP (Model Context Protocol): "Think of it as plugins for AI. Connect Claude to your database, your CRM, your Slack — it can read and write to any of them."
  • CLIs: "Claude Code runs in the terminal. Why? Because that's where all the power tools live. Git, databases, file systems — Claude can use all of them directly."
  • CLAUDE.md: "This is like an instruction manual for your AI. You put it in your project and Claude reads it every time. 'Always use Supabase. Always add tests. Use this color palette.' It remembers your preferences."
  • Multi-session: "Run multiple Claude instances on different tasks simultaneously. While one is building your frontend, another is setting up your database."

Slide 30 — Pricing

Say: "Let's talk money. Claude Max is $100-200/month depending on the tier. That sounds like a lot until you realize a junior developer costs $6-8K/month. A freelancer costs $100-150/hour. I'm building production apps for the cost of a nice dinner."

"And there are free tiers for most tools. You can start with $0 today."
